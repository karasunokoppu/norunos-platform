1. プロジェクト概要
   - 目的・概要:
     - タスクやメモの管理、収集機能を搭載したアプリ
     - できるだけ低い占有メモリを目指す
   - 対象ユーザー: 個人用
   - 想定ユーザー数: 数名
   - 提供プラットフォーム: PC (オプションとしてAndroidも)

2. システム構成
   - Tauri2.0 (React + TypeScript + Rust)

   - データ保存方法
     - SQLite (ToDoList, カレンダー[Daily Memo含む], ガントチャート)
     - ファイル(ファイルのメタ情報はデータベース管理) (Markdownメモ: `Documents/NorunosNotes`, マインドマップ)

   - 将来のデバイス間データ同期を想定した実装
     - 全エンティティ(各タスクとか)にUUID/ULIDを付与(SQLiteのAUTOINCREMENTは使用しない)
     - 更新管理データ(created_at, updated_at, deleted_at(nullable))を全エンティティにもたせる

   2-2. デバイス間データ同期
      - データ保存ロジックはすべてRust側で実装
      - コンフリクト(競合)の解決方法は最終更新を優先
      - データベースのスキーマのバージョン管理
      - 実装ロードマップ
         1. 同期なし（ローカル完成）
         2. ZIPエクスポート
         3. 同期ディレクトリ指定
         4. WebDAV対応
         5. 暗号化オプション
         6.（必要なら）サーバー

3. 機能仕様
   - 主要機能一覧
     - To Do List (タスク管理, グループ化)
     - カレンダー (期日別タスク表示, 日別メモ[📝アイコン/内容表示])
     - ガントチャート (タスクグループ分け, "今日"ハイライト表示)
     - Markdownメモ (フォルダ階層表示, 分割エディタ, ローカルファイル保存)
     - リーディングメモ
     - マインドマップ
   - アプリの非起動時(使用時)に複数のデバイス間でのデータ同期機能を実装(将来的には)

4. アカウントとセキュリティ要件
   - アカウント: OSのログインアカウントでの管理とする
   - セキュリティ: アプリ起動時のパスワード入力要求を実装(将来的には)
