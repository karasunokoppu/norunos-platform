1. プロジェクト概要
   - 目的・概要: 
     - タスクやメモの管理、収集機能を搭載したアプリ
     - できるだけ低い占有メモリを目指す
   - 対象ユーザー: 個人用
   - 想定ユーザー数: 数名
   - 提供プラットフォーム: PC (オプションとしてAndroidも)

2. システム構成
   - Tauri2.0 (React + TypeScript + Rust)

   - データ保存方法
     - SQLite (ToDoList, カレンダー, ガントチャート)
     - ファイル(ファイルのメタ情報はデータベース管理) (Markdownメモ、　マインドマップ)

   - 将来のデバイス間データ同期を想定した実装
     - 全エンティティ(各タスクとか)にUUID/ULIDを付与(SQLiteのAUTOINCREMENTは使用しない)
     - 更新管理データ(created_at, updated_at, deleted_at(nullable))を全エンティティにもたせる

   2-2. デバイス間データ同期
      - データ保存ロジックはすべてRust側で実装
      - コンフリクト(競合)の解決方法は最終更新を優先
      - データベースのスキーマのバージョン管理
      - 実装ロードマップ
         1. 同期なし（ローカル完成）
         2. ZIPエクスポート
         3. 同期ディレクトリ指定
         4. WebDAV対応
         5. 暗号化オプション
         6.（必要なら）サーバー

1. 機能仕様
   - 主要機能一覧
     - To Do List
     - カレンダー
     - ガントチャート
     - Markdownメモ
     - リーディングメモ
     - マインドマップ
   - アプリの非起動時(使用時)に複数のデバイス間でのデータ同期機能を実装(将来的には)

2. アカウントとセキュリティ要件
   - アカウント: OSのログインアカウントでの管理とする
   - セキュリティ: アプリ起動時のパスワード入力要求を実装(将来的には)

│   │   │   ├── mod.rs
│   │   │   ├── commands.rs
│   │   │   └── service.rs   # ファイル読み書きロジック
│   │   └── mindmap/         # マインドマップ関連 (ファイル操作)
│   │       └── ...
│   │
│   ├── infra/               # インフラ層（DB接続や共通処理）
│   │   ├── mod.rs
│   │   ├── db.rs            # SQLite接続・初期化・マイグレーション
│   │   └── fs_utils.rs      # ファイルパス解決などのヘルパー
│   │
│   ├── shared/              # アプリ全体で使う型やエラー定義
│   │   ├── mod.rs
